# hmi_server 具体代码逻辑

> 本文档描述 hmi_server（人机交互进程）的内部代码逻辑，包括启动流程、线程模型、按键检测、LED 状态机、命令仲裁、通信交互与状态转换，不含源码。

---

## 1. 启动流程

### 1.1 HMI 对象创建与启动
实例化 HMI 核心对象并调用 Start 方法。Start 会完成 LCM 订阅注册、业务配置参数加载，并创建一个名为 t1 的工作线程进入主循环。

### 1.2 RPC 服务器初始化
在端口 9002 上启动基于 rest_rpc 的服务器，注册六个远程调用接口，分别对应 App 命令、基站命令、按键命令、按键码、以及两类原始命令的接收。

### 1.3 运行时环境设置
将当前进程线程命名，并绑定到指定的 CPU 核心上运行。

### 1.4 主循环进入
设置一个运行标志，启动一个 200 毫秒周期的循环。循环内部依次执行：
- 通过 TCP 向 navclean 进程查询当前状态；
- 从 HMI 内部的 App 命令队列中取出命令，映射为文本指令转发给 navclean；
- 处理原始 App 命令和原始导航命令；
- 检测 HMI 内部机器人状态的变化，若发生有效状态跃迁，则向 navclean 发送对应的动作指令；
- 休眠后继续下一轮。

当收到退出指令时，先停止 HMI 内部线程，再关闭 RPC 服务器，最后延迟 3 秒后进程退出。

## 2. 核心线程模型

hmi_server 采用双线程协作模型：

### 2.1 t1 线程（内部核心循环）
由 Start 创建，以 100 赫兹（每 10 毫秒）的频率运行。其职责为：
- **按键扫描与解码**：周期性读取 GPIO 设备，检测物理按键动作，解码为具体按键命令。
- **命令处理**：依次处理按键命令、基站命令、App 命令，每个命令都会进行当前状态合法性检查。
- **LED 驱动更新**：根据当前机器状态计算 LED 显示模式，并输出到 GPIO。
- **LCM 消息分发**：调用 LCM 的超时处理函数，接收来自导航、地图等模块的状态更新。
- **GPIO 输出**：将计算好的电源控制位和 LED 控制位写入硬件设备。
- **遥控速度发布**：若处于遥控模式，按 10 赫兹向导航模块发布线速度和角速度指令。
- **定时广播**：每 100 毫秒通过 LCM 向全系统广播机器人状态；每 120 毫秒向 App 模块推送机器人信息。

### 2.2 main 函数线程（外部交互循环）
以 5 赫兹（每 200 毫秒）运行，职责为：
- 作为与 navclean 进程的 TCP 客户端，持续查询状态并转发上层命令。
- 作为 RPC 服务器端，响应来自 app_server 的远程调用。
- 监控 HMI 内部状态机的变化，驱动 navclean 执行实际动作。

两个线程之间通过 HMI 对象的成员变量共享状态，当前实现未使用显式互斥锁，依赖单生产者单消费者的时序约束。

## 3. 按键检测逻辑

按键检测基于对 GPIO 字符设备的轮询读取，检测两个物理按键：电源键和功能键。

### 3.1 检测时序原理
系统为每个按键维护独立的持续计数器，阈值定义如下：
- **短按**：按键持续超过约 0.1 秒后释放。
- **长按**：按键持续超过约 3 秒，或按下期间达到 3 秒时触发。
- **组合键**：两键同时按下超过约 3 秒。

### 3.2 识别流程
每次读取按键寄存器后，按以下优先级判定：
1. 若两键同时处于按下状态，累加组合计数器；当达到长按阈值，标记为组合长按。
2. 若仅电源键按下，累加电源计数器；释放时根据持续时间判定为电源长按或电源短按。若按下期间达到阈值，则提前标记为电源长按。
3. 若仅功能键按下，同理判定为功能键长按或功能键短按。

返回结果的优先级严格为：组合长按 > 电源长按 > 功能键长按 > 电源短按 > 功能键短按。

### 3.3 按键命令解码
检测到的按键码会根据当前机器人状态机翻译为实际命令：
- **休眠状态**下，任何有效按键均翻译为唤醒命令。
- **待机状态**下，电源短按翻译为开始清扫；功能键短按翻译为回充；组合长按翻译为重置 WiFi。
- **工作状态**下，电源短按或功能键短按均翻译为暂停。
- **暂停状态**下，电源短按通常翻译为继续清扫，但在充电且处于特定外部任务时，会转为回到待机并重新启动清扫。
- **任意状态**下，电源长按翻译为关机。

## 4. LED 状态机

LED 显示模式由一个独立的状态机驱动，其状态取值包括：工作、暂停、待机、休眠、错误、回充中、与基站协同工作中、充电中、充电完成。

### 4.1 状态判定优先级
1. **最高优先级**：若机器人被标记为已停止（硬件错误或导航错误），强制进入错误模式。
2. 正常情况下根据主状态机取值：
   - 休眠 → 熄灭
   - 待机 → 白灯常亮
   - 暂停 → 白灯常亮
   - 工作状态下进一步细分：
     - 若工作模式为回充且未在充电 → 白灯慢闪（回充中）
     - 若工作模式为与基站协同 → 绿灯慢闪
     - 若正在充电 → 白灯慢闪（充电中）
     - 其他正常工作 → 白灯常亮

### 4.2 驱动实现
LED 通过 GPIO 的三色引脚控制。各种闪烁模式通过静态计数器实现周期性开关：
- **红灯快闪**：约 16 周期亮、17 周期灭，形成快速红色闪烁，用于错误报警。
- **白灯慢闪**：约 50 周期灭、50 周期亮，用于回充和充电提示。
- **绿灯慢闪**：周期与白灯慢闪相同，仅颜色通道切换为蓝绿混合，用于基站协同工作提示。

## 5. 命令仲裁逻辑

系统存在三类命令源，它们以串行顺序在 t1 线程的每个周期内被处理，没有显式的抢占锁，而是依靠状态机条件检查实现互斥。

### 5.1 三类命令源
1. **按键命令**：由物理按键或 RPC 接口注入。
2. **基站命令**：由基站面板或 RPC 接口注入。
3. **App 命令**：由移动端或 RPC 接口注入。

### 5.2 仲裁机制
在每个 10 毫秒周期内，t1 线程按以下顺序处理：先处理按键命令，再处理基站命令，最后处理 App 命令。

每个处理函数内部都会检查当前状态机是否满足该命令的前置条件。例如：
- 开始清扫只允许在待机或休眠状态下执行；
- 暂停只允许在工作状态下执行；
- 继续只允许在暂停状态下执行；
- 回充要求当前不在充电状态。

若条件不满足，命令会被拒绝，记录错误日志，并将命令变量重置为空。若多个命令在同一周期到达且状态条件允许，后处理的命令会覆盖先处理命令导致的状态变化。由于命令处理周期极短（10 毫秒），实际运行中通常表现为最近一次有效输入胜出。

## 6. 与 navclean 的 TCP 文本交互

hmi_server 作为 TCP 客户端，通过本机 9001 端口与 navclean 进程通信，采用一问一答的文本协议。

### 6.1 通信格式
- **请求**：纯文本字符串，以换行符结尾。支持的指令包括状态查询和动作指令。
- **响应**：navclean 返回的文本字符串。状态查询响应包含充电标志和错误标志等键值对。

### 6.2 交互内容
主循环中每 200 毫秒执行以下 TCP 交互：
1. **状态查询**：发送固定查询指令，解析响应中的充电字段更新机器人充电标志，解析错误字段更新停止标志。
2. **命令转发**：将来自 App 的布尔型命令按数据点 ID 映射为文本动作指令，包括开始/停止清扫、暂停/继续、开始/停止回充、开始/停止建图、开始/停止洗拖布并回充。
3. **状态变化转发**：监控 HMI 内部状态机的变化，若检测到有效跃迁（如待机到工作、工作到暂停、暂停到工作、工作/暂停到待机），则自动向 navclean 发送对应的动作指令，确保 HMI 状态与 navclean 实际行为同步。

## 7. 与 app_server 的 RPC 交互

hmi_server 作为 RPC 服务端，在 9002 端口为 app_server 提供以下接口：

### 7.1 枚举型命令接口
- **SetAppCmd**：接收移动端发来的应用级命令（如开始全屋清扫、开始划区清扫、暂停、继续、停止、回充等），直接写入 App 命令变量。
- **SetStaCmd**：接收基站发来的命令（如基站面板启动清扫、暂停、继续、出桩、回充等），写入基站命令变量。
- **SetKeyCmd**：模拟按键命令（唤醒、开始清扫、暂停、继续、回充、关机），写入按键命令变量。
- **SetKeyCode**：直接注入按键码（电源短按/长按、功能键短按/长按、组合长按），供调试或测试使用。

### 7.2 数据点命令接口
- **RpcApi_APPCMD**：接收图雅数据点格式的命令。对布尔型数据点进行解析：数据点映射为开始/停止清扫、暂停/继续、回充/停止、遥控方向指令、地图重置、洗拖布并回充、LED 开关、App 连接心跳等。解析后的命令放入内部队列，由主循环转发给 navclean。
- **RpcApi_APPRAWCMD**：接收原始字节流命令，处理复杂业务：选区清扫（解析房间编号序列）、定点清扫（解析虚拟坐标）、划区清扫（解析多个矩形区域顶点坐标）、勿扰时间设置、特殊房间清扫等。

## 8. LCM 状态广播

HMI 模块通过 LCM 消息总线与系统内其他模块进行状态同步。

### 8.1 对外广播
- **频道名**：机器人状态广播通道。
- **消息内容**：时间戳、当前状态机值、当前工作模式、机器人是否停止标志。
- **发布频率**：在 t1 线程中约为 10 赫兹；在关键状态转换点（如开始清扫、停止、回充启动）会立即额外发布一次，确保其他模块实时感知。

### 8.2 其他 LCM 通信
- **向 App 推送**：每约 8.3 赫兹（周期 120 毫秒）在专用通道上发布完整的机器人信息结构，包含电池电量、清扫面积、清扫时间、错误码、基站状态等。
- **向导航发遥控速度**：在遥控模式下以 10 赫兹发布速度指令。
- **接收导航状态**：订阅导航到 HMI 的状态通道，从中提取充电状态、错误码、外部任务标志、清扫完成标志等，用于修正 HMI 内部状态机。
- **接收地图信息**：订阅地图信息通道，根据地图宽高判断是否存在有效地图，用于决定 App 建图命令的实际行为（无图时启动建图，有图时启动清扫）。
- **接收调试指令**：订阅调试命令通道，支持通过 LCM 注入模拟的 App 命令。

## 9. 状态转换图

机器人主状态机包含五个核心状态：休眠、待机、工作、暂停、关机。

### 9.1 休眠状态
- 任意按键短按 → 待机：物理按键唤醒。
- 长时间静止超时 → 休眠：当机器处于待机或暂停状态超过 300 秒无变化，自动进入休眠。

### 9.2 待机状态
- 电源键短按 / App 开始清扫 / 基站开始清扫 / 调试指令 → 工作：进入清扫工作模式。
- 功能键短按 / App 回充 / 基站回充 → 工作（回充模式）：前提是不在充电状态。
- 电源键长按 → 关机。

### 9.3 工作状态
- 电源键短按 / 功能键短按 / App 暂停 / 基站暂停 → 暂停。
- App 停止 / 导航报告清洁完成 / 导航报告出桩完成 → 待机。
- 导航报告错误停止（硬件错误、激光错误、电池错误） → 暂停：同时标记机器人已停止。
- 内部工作模式切换：可在全屋清扫、划区清扫、选区清扫、定点清扫、快速建图、遥控、回充、自定义清扫、洗拖布回充等子模式间切换。

### 9.4 暂停状态
- 电源键短按 / App 继续 / 基站继续 → 工作：通常恢复为全屋清扫模式。
- App 停止 / 清洁完成 / 回充到达充电状态 → 待机。
- 导航报告错误停止 → 暂停保持：错误未解除时维持暂停。

### 9.5 关机状态
为终止状态，进入后向 GPIO 写入复位信号，切断主控电源。

### 9.6 特殊状态修正
- **充电状态同步**：当通过 TCP 或 LCM 确认机器人处于充电状态且持续一段时间后，若当前为暂停状态，会强制转为待机状态。
- **低电量回充**：当工作状态下检测到电池电量低错误，标记低电量回充标志，以便后续按键解码逻辑做出特殊处理。
- **清洁完成检测**：通过导航模块的外部任务标志或清扫完成标志双重检测清洁完成事件，完成后转入待机并向 App 发送完成通知。
- **激光/电池严重错误**：检测到特定致命错误位时，强制将状态机切入暂停，并点亮错误 LED。
