# platform 具体代码逻辑

> 本文档描述 platform（硬件抽象层）的内部代码逻辑与 MCU 通信协议，包括 XBase_t 架构、串口帧格式、传感器处理、执行器控制、基站通信、电源管理和 LCM 交互，不含源码。

---

## 1. XBase_t 整体架构

### 1.1 类职责定位
XBase_t 是机器人硬件抽象层的核心单例类，承上启下：
- **对下**：通过串口与 MCU 进行双向数据交换，管理激光雷达（X1C/XD1Q）、XD1Y（ToF/结构光）、I2C 设备等传感器的启停与数据读取。
- **对上**：通过 LCM 消息总线向导航、SLAM、APP 等上层模块发布传感器数据，并订阅上层下发的速度指令、执行器指令等。

### 1.2 初始化流程
构造函数阶段主要完成：
- 从配置文件中加载参数（加速度限制、悬崖阈值、激光安装位姿、各传感器使能开关等）。
- 初始化各类状态变量、错误码、滤波器窗口、电量映射表、历史数据缓存区等。
- 保存 LCM 实例指针，供后续消息收发使用。

Init 方法按以下顺序启动硬件：
1. **加载参数**。
2. **初始化 XD1Y**：默认不立即打开，仅加载标定文件路径；在需要时由外部调用真正启动。
3. **初始化 MCU 串口**：打开 UART 并设置标志位。
4. **注册 LCM 回调**：订阅速度指令、电机占空比、毛发切割、定位结果、SLAM 位姿、遥控等通道。
5. **初始化 PCB 按键**：当前实现为空，由 APP 层处理。
6. **初始化 XD1Q 障碍物检测参数**：根据安装高度和俯仰角预计算参考数组和阈值数组。
7. **初始化激光雷达数据结构**：分配 X1C、XD1Q、XD1Y 的点云/强度数组，并计算各角度的三角函数查找表。
8. **设置运行标志**，并初始化默认风机占空比。

### 1.3 主循环线程模型
XBase_t 不自行创建线程，其生命周期由外部任务线程驱动（通常以 20ms 为周期循环调用）。一个完整周期分为两大阶段：

**输入更新阶段**：
- 调用 MCU 更新函数从串口环形缓冲区中解析 MCU 上行帧。
- 若解析成功，内部依次执行：碰撞状态更新、IMU 与编码器融合、悬崖检测、四元数转 RPY、消息打包。
- 若 XD1Y 已打开，从 I2C 线程的队列中取最新原始数据，经标定转换后得到距离和强度，并计算等效墙检距离。

**输出更新阶段**：
- 若激光打开，调用底层接口读取 X1C/XD1Q 原始数据，转换为点云，执行 XD1Q 障碍物检测。
- 执行电量平滑滤波。
- 执行打滑检测。
- 执行拖地卡死恢复。
- 执行 MCU 下行指令封装与发送。
- 执行 LCM 消息派发。
- 执行 LCM 消息发布。
- 记录轨迹与统计清扫面积。

时序保护：若本次周期耗时超过 20ms 容限，则跳过 LCM 处理并报警；若耗时不足，则主动睡眠补足周期。

## 2. MCU 串口通信协议详解

### 2.1 帧格式
采用固定长度、定界符包裹的同步帧结构：
- **帧头**：四字节固定魔数。
- **帧尾**：四字节固定魔数。
- **数据域**：帧头与帧尾之间为完整结构化数据，长度固定，无转义。

校验方式：上行帧解析仅检查帧头和帧尾是否同时匹配，若头尾不对应，则丢弃整帧并打印异常。下行帧由发送端直接填充头尾后写出。

### 2.2 上行数据（MCU → 主机）
MCU 以固定周期发送状态帧，包含以下字段类别：

**电机与运动反馈**：
- 左右轮电机 PWM 占空比反馈、电流、编码器脉冲累计值、温度。
- 风机 PWM 占空比反馈、电流、脉冲（转速反馈）。
- 轮速环反馈：线速度、角速度。
- 里程计输出：X、Y 坐标、Yaw 角。

**IMU 数据**：
- 四元数。
- 附加角度与角速率。
- 原始三轴加速度计与三轴陀螺仪数据。

**传感器与状态**：
- 悬崖传感器 6 路 ADC 值。
- 电池电压 ADC、电池 NTC ADC。
- 碰撞传感器：左碰撞、右碰撞。
- 尘盒缺失、水箱缺失、充电中、碰撞处理中、MCU 抖动等错误位域。
- 主刷升降电机限位开关、硬地板检测、电池在位状态、充电完成状态。
- 扩展状态字节：右边刷霍尔、拖地左右霍尔、IMU 需停止标志、拖地左右红外。
- 固件版本号。

**RF 基站数据**：
- 嵌套在 MCU 帧中的 RF 数据结构，包含基站 tick、收发地址、以及基站回传的状态位。

### 2.3 下行数据（主机 → MCU）
主机以固定周期发送控制帧，包含以下字段：

**运动指令**：
- 目标线速度、目标角速度。
- 左右轮电机 PWM 占空比。

**执行器指令**：
- 风机占空比。
- 右刷占空比、主刷占空比。
- 拖地电机占空比、主刷升降电机占空比。

**命令字（位域）**：
- 速度环使能、轮电机使能、清除错误、里程计复位、碰撞手动后退模式。
- RF 响应使能、超声波调试。

**RF 数据透传**：
- 将主机要发给基站的 RF 数据直接嵌入 MCU 帧，由 MCU 通过射频转发。

## 3. 传感器数据处理流程

### 3.1 激光雷达（X1C / XD1Q）

**启动/关闭**：
- 打开激光传感器函数在特定构建目标下调用底层打开 XD1Y 与打开激光函数；若失败记录错误标志。
- 关闭激光传感器函数调用底层关闭激光与关闭 XD1Y。

**X1C 原始数据解析为点云**：
- 底层返回整型数组，每个元素代表一个激光点的距离，单位为 mm。
- 在输出更新阶段，遍历 1200 个点：两端各 100 点强制置 0；若原始值为特殊标记，置 0 并计数；其余点转换为米并限制最大距离；若无效点超过总数 1/3，则整帧清零。

**XD1Q 处理**：
- 底层返回 680 个原始距离值。
- 先进行基础滤波：过小或过大的原始值视为无效。
- 连续性滤波：遍历相邻点，差值过小的视为同一连续段；连续有效点不足的段全部置 0；连续段长度在特定范围时，根据弧长计算障碍物宽度，宽度过小的视为噪声并清除。
- 坐标转换：利用预计算的三角函数查找表以及安装俯仰角，将极坐标转换为三维点云。

### 3.2 里程计

**编码器计数转换为位姿**：
- MCU 内部维护里程计，通过上行帧直接输出 X、Y 和 Yaw。
- 消息打包函数将其转换为米和弧度，包装为里程计消息。
- 角度归一化到合理范围。

**中值滤波**：
- 维护最近 3 帧里程计的历史窗口。
- 计算三帧之间两两的角度差，取角度变化最小的一帧作为输出。
- 若当前帧与上一帧输出差异极大，打印错误日志。

**打滑检测**：
- 计算左右轮脉冲差值的变化量，并与 IMU 输出的 Yaw 角变化量对比。
- 计算比率，若比率小则衰减错误计数，若比率大则按阈值累加错误计数。
- 当计数超过阈值时，触发打滑标志，向上层发送重定位请求。

### 3.3 IMU

**四元数解析**：
- MCU 上行四元数按固定缩放系数解析。
- 四元数转 RPY 函数先将四元数归一化，再利用标准公式转换为 Roll、Pitch、Yaw。
- 对万向节锁附近做特殊处理。

**Yaw 角融合**：
- 内部维护两种更新模式：
  - 模式一：基于编码器脉冲差和 IMU Yaw 输入，检测脉冲差变化时 IMU 是否同步更新，若连续多帧无更新则报错。
  - 模式二：基于陀螺仪 Yaw 与融合 Yaw，若陀螺仪变化超过阈值则更新融合 Yaw；并监控陀螺仪与融合 Yaw 的差值，若超过阈值持续异常则报错。

**俯仰角监控**：
- 实时 Roll/Pitch 通过滑动平均加稳态检测滤波器进行平滑。
- 若 Roll 或 Pitch 超过阈值持续一定时间，置位不平衡错误码。

### 3.4 碰撞 / 悬崖 / 墙检

**碰撞**：
- 原始状态来自 MCU 错误位域的左右碰撞标志。
- 碰撞状态更新函数跟踪碰撞处理状态：若当前处于处理中，记录左右碰撞标志；处理结束后，将最终碰撞状态保存。
- 对外暴露的碰撞检测函数综合了物理碰撞、悬崖碰撞处理中、以及虚拟碰撞。

**悬崖**：
- 6 路悬崖传感器 ADC 值。
- 悬崖更新函数将每路 ADC 与阈值比较，生成原始悬崖位图。
- 每路独立计数器进行滤波：触发时递增，未触发时减半；超过窗口大小后置位悬崖状态。
- 悬崖使能开关可禁用此功能；禁用时强制将所有悬崖 ADC 设为无效高值。

**墙检**：
- 墙检距离由 XD1Y 处理后的数据或 MCU 上行帧的距离数据提供。
- 单位为 mm，对外转换为米。

### 3.5 XD1Y（ToF / 结构光）

**I2C 通信**：
- 打开 XD1Y 函数打开 I2C 总线，读取模块 SN 号，写入寄存器启动测量。
- 创建独立抓取线程，循环读取状态寄存器：若表示有新数据，批量读取原始数据压入队列；若表示模块异常，重新写启动命令。
- 更新 XD1Y 函数从队列中取出最新一帧原始数据。

**原始数据处理**：
- 使用标定文件进行查表和插值：原始数据分为上下两半，前半为距离索引，后半包含子像素、置信度和强度信息。
- 根据距离索引和列号查标定表得到角度和距离映射参数。
- 将原始列数据映射到标准角度桶中，对空缺角度进行线性插值填充。
- 输出距离加置信度数据和强度数据。
- 进一步做中值滤波和分段均值滤波，然后按安装高度和俯仰角筛选地面以上有效点，返回最小水平距离作为等效墙检值。

## 4. 执行器控制流程

### 4.1 速度指令转换为电机 PWM
- 上层通过 LCM 的速度指令通道下发目标线速度和角速度。
- 速度命令更新函数接收后存入目标速度，并重置速度超时计时器。
- 控制 MCU 函数中：
  - 若速度超时，自动清零速度。
  - 若使能速度规划，对目标速度进行加速度限幅（梯形速度规划）。
  - 若虚拟碰撞触发，强制后退。
  - 若悬崖使能，根据悬崖位图覆盖速度指令（后退/旋转/停止）。
  - 最终将目标速度转换为毫米级和毫弧度级写入下行帧。

### 4.2 拖地 / 主刷升降控制

**拖地升降**：
- 拖地升起/放下函数设置指令标志。
- 控制 MCU 函数中非回充模式下：若指令为升起，检测拖地左右红外；若未到位，输出固定占空比持续最多 5 秒；到位后停转。若指令为放下，由外部直接控制占空比。

**主刷升降**：
- 设置主刷升降方向函数设置方向。
- 控制 MCU 函数中根据限位开关和运行时间保护输出占空比；超时后停止并报错。

### 4.3 风机转速控制
- 设置风机占空比函数直接设置目标占空比。
- 清扫模式下：若目标占空比大于阈值，采用平滑过渡策略，从基础值开始以固定速率缓慢爬升到目标值，避免风机瞬间全速启动。否则直接等于目标值。
- 非清扫模式下强制清零。

## 5. XStation_t 基站通信

### 5.1 RF 数据包格式
RF 数据结构嵌入在 MCU 上行/下行帧中，字段如下：
- **tick 与地址**：基站 tick、基站低位/高位地址、机器人低位/高位地址。
- **机器人发往基站**：
  - 电机域：清水泵功率、清洁液泵功率。
  - IO 域：回家灯、PTC 模块、集尘、静音、注水阀、排水泵、风机、污水泵、机器人在站标志、满充标志、关闭面板 LCD、童锁等开关位。
  - 机器人状态域：运行工况、工作模式、工作类型。
- **基站发往机器人**：
  - IO 域：清水箱在位/空/满、污水箱在位/满、清洁液箱在位/空、尘盒在位、清洗盘在位/水满、PTC 状态、回家灯、阀门状态、静音、电源状态等。
  - 工作状态域：基站状态、面板控制消息、面板灯、童锁。
  - 版本域：基站版本号。
- 尾部跟随两字节校验和。

### 5.2 基站状态解读
基站类提供解析接口，将基站回传数据的位域转换为布尔状态或枚举：
- 各类水箱/尘盒/清洁液盒：在位检测、空/满检测。
- 基站运行状态：电源开启、洗拖布、集尘、烘干、湿拖、进入基站。
- 面板按键消息：启动单扫、单拖、扫拖、自定义、停止、继续、出基站、回充等，映射为基站命令枚举。

### 5.3 双向通信流程
- **下行**：上层逻辑通过基站类的设置方法填充发往基站的 RF 数据，XBase_t 在控制 MCU 函数中将其复制到 MCU 下行帧，并置位 RF 响应使能。MCU 收到后通过射频发给基站。
- **上行**：MCU 将基站回传的 RF 包嵌入上行帧；XBase 取出并交给基站类的解析函数，上层再调用各类查询方法查询状态。

## 6. 虚拟碰撞与 3D 障碍物检测

### 6.1 虚拟碰撞状态机
平台层维护两类虚拟碰撞：
- **LocalMap 虚拟碰撞**：由设置函数触发，执行固定时长的后退。当前实现中未实际启用。
- **XD1Q 3D 障碍物检测**：在输出更新阶段，若 XD1Q 数据更新成功，调用障碍物检测函数。检测前提：Roll/Pitch 滤波器处于稳态且硬地板检测为真；否则认为机身姿态不稳，放弃检测。

### 6.2 XD1Q 3D 障碍物检测详细流程
- **分段与滤波**：遍历 XD1Q 点，筛选出符合 x、z、y 条件的点，聚类为障碍物段。仅保留长度足够大的段。
- **最近障碍物提取**：在所有有效段中找中心点 x 距离最小且在 y 范围内的段作为最近障碍物。记录触发时刻的里程计、障碍物距离、方向角。触发后自动将目标线速度减半。
- **状态锁定**：一旦置位，在机器人运动且检测使能时保持锁定，直到外部调用清除函数。

### 6.3 3D 障碍物使能栈
支持将当前检测使能状态压栈和弹出恢复。用于临时禁用 3D 检测（如过门槛时）后恢复。

## 7. 电源管理

### 7.1 电量平滑滤波
电量计算函数实现多阶段电量估计：

1. **ADC 读取与线性映射**：读取 MCU 上行电池 ADC，先乘以修正系数，再映射到压缩区间。若 ADC 异常持续多帧，报电池 ADC 错误。

2. **滑动平均滤波**：固定长度滑动窗口对压缩后的 ADC 值求平均，得到平滑平均值。

3. **偏置修正**：根据当前风机占空比向上修正（风机负载越大，ADC 读数越虚高）；若正在充电，再向下修正。

4. **查表与充放电速率限制**：通过预置的电量映射表将平滑 ADC 转换为电量百分比。
   - **放电限制**：非充电状态下，输出电量只能以固定速率下降，不能跳变。
   - **充电限制**：充电状态下，以固定速率上升。
   - 此设计防止电量数字因负载波动而剧烈跳动。

5. **低电量告警**：若输出电量低于阈值，置位低电量标志。

### 7.2 充电状态检测
- 直接读取 MCU 上行错误位域的充电标志。
- 充电完成通过充电完成位域获取。
- 机器人在站函数返回充电中或充电完成任一状态为真。

## 8. LCM 发布/订阅交互

### 8.1 订阅通道（上层 → XBase）
在设置消息回调函数中注册以下 LCM 回调：

| 通道 | 作用 |
|---|---|
| 速度指令 | 接收导航层下发的目标速度，支持速度环使能开关和里程计复位请求 |
| 电机占空比指令 | 接收各电机的占空比指令 |
| 毛发切割指令 | 毛发切割使能 |
| AMCL 里程计 | 接收 AMCL 定位输出的位姿 |
| SLAM 位姿信息 | 接收 SLAM 位姿信息 |
| 调试指令 | 接收遥控/调试指令 |

### 8.2 发布通道（XBase → 上层）
在消息发布函数中按周期或事件发布：

| 通道 | 触发条件 |
|---|---|
| 激光数据 | X1C 数据更新时 |

### 8.3 LCM 句柄处理
- LCM 处理函数以非阻塞方式循环处理所有待接收的 LCM 消息直到队列为空。
- 若激光处理阶段耗时超过阈值，则跳过 LCM 处理，防止周期膨胀。
