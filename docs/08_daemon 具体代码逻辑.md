# daemon 具体代码逻辑

> 本文档描述 daemon（日志守护进程）的内部代码逻辑，包括启动流程、配置读取、主循环、日志轮转与心跳保活机制，不含源码。

---

## 1. 总体架构

该守护进程是一个极简的单文件进程，核心职责是：以固定周期调用系统自带的 logrotate 工具进行日志轮转，同时通过周期性日志输出证明自身存活。整个程序不包含多线程、网络通信或复杂状态机，逻辑集中在配置加载、日志初始化、无限循环三个环节。

---

## 2. 启动流程与初始化

### 2.1 确定配置目录
首先尝试从环境变量读取配置根目录。若该环境变量未设置或为空字符串，则回退到默认路径。

### 2.2 加载自身配置
调用配置加载函数，构造配置文件完整路径，随后解析该 INI 文件。若文件不存在或解析异常，则全部使用硬编码的默认值并输出错误提示。

### 2.3 参数校验与修正
对读取到的配置进行两项边界校验：
- **轮转周期**：若小于等于 0，则强制修正为 30 分钟。
- **日志级别**：若超出有效范围，则强制修正为信息级。

### 2.4 初始化日志系统
创建一个日志对象，传入是否启用日志开关；随后调用初始化接口，指定守护进程自身的日志文件名，并根据校验后的结果设置日志输出阈值。

### 2.5 记录启动标记
向日志文件写入一条信息级记录，标志守护进程已启动。

### 2.6 预计算 logrotate 命令字符串
根据配置目录，拼接出待执行的系统命令字符串。

### 2.7 初始化时间戳基准
获取当前系统时间戳（毫秒级），分别作为心跳计时基准和轮转计时基准的初始值。

---

## 3. 配置读取逻辑

守护进程涉及两个配置文件，用途和解析方式各不相同。

### 3.1 守护进程自身行为配置
该文件决定守护进程多久执行一次日志轮转以及日志系统如何工作。

**文件格式**：标准 INI 格式，包含一个名为 Interval 的节。

**解析的键值对**：

| 键名 | 含义 | 数据类型 | 缺失或异常时的行为 |
|------|------|----------|-------------------|
| 是否启用日志 | 是否启用日志输出 | 布尔型 | 默认启用 |
| 日志级别 | 日志输出阈值级别 | 整型 | 默认信息级 |
| 是否启用定时轮转 | 是否启用定时轮转 | 布尔型 | 默认不启用 |
| 轮转周期 | 轮转周期（分钟） | 整型 | 默认 30 分钟 |

**解析细节**：
- 使用项目自研的 INI 解析器读取。
- 每个键值通过模板函数按目标类型转换；转换失败时捕获异常，打印警告，并保留默认值。
- 若文件整体不存在，直接返回全部默认值，不做任何解析。

### 3.2 logrotate 的配置文件
该文件并非由守护进程解析，而是作为参数直接传递给外部 logrotate 命令。守护进程仅负责拼接出该文件的绝对路径，并检查其存在性（实际上在构造命令时并不检查存在性，执行时由 logrotate 自行处理）。

---

## 4. 主循环详细逻辑

进入主循环后，程序以固定 5 秒为周期反复休眠和唤醒。每次唤醒后执行以下两个检查：

```
[开始]
  │
  ▼
[获取当前时间戳]
  │
  ├──→ 检查心跳计时器 ──→ 若距离上次心跳已 ≥ 60 秒？
  │       │                        │
  │       否                      是
  │       │                        ▼
  │       │              [更新心跳时间戳 = 当前时间]
  │       │                        │
  │       │                        ▼
  │       │              [写入 Info 级日志：daemon is working]
  │       │
  └──→ 检查轮转计时器 ──→ 若"启用轮转"为真 且 距离上次轮转已 ≥ 配置周期？
          │                        │
          否                      是
          │                        ▼
          │              [更新轮转时间戳 = 当前时间]
          │                        │
          │                        ▼
          │              [写入 Debug 级日志：rotate log once]
          │                        │
          │                        ▼
          │              [执行 logrotate 命令]
          │                        │
          │                        ▼
          │              [检查命令返回值]
          │                        │
          │              返回非 0 ──→ [写入 Warn 级日志，记录命令与返回码]
          │
          ▼
    [休眠 5 秒]
          │
          └──→ 回到循环开始
```

### 4.1 休眠周期
每次循环末尾固定休眠 5 秒，这意味着心跳和轮转的检查精度都是 5 秒级别。

### 4.2 心跳检查
与轮转检查相互独立，即使日志轮转未启用，心跳仍每 60 秒执行一次。

### 4.3 轮转检查
只有当启用轮转标志为真时才会触发；若关闭轮转功能，则仅执行心跳和休眠。

---

## 5. 日志轮转机制

### 5.1 触发条件
主循环中，同时满足以下两点时触发：
1. 配置项 intervalEnable 为真；
2. 当前时间与上次轮转时间的差值大于等于配置的分钟数。

### 5.2 命令构造方式
在启动阶段预先生成完整的 shell 命令字符串，格式为：
```
logrotate {配置目录}/logconfig.cfg
```
其中配置目录来自环境变量或默认路径。

### 5.3 执行方式
通过标准库的系统调用接口直接执行上述命令，等命令完成后获取其返回码。

### 5.4 错误处理
若 logrotate 返回非 0（表示执行失败），守护进程不会终止，而是向自身日志写入一条警告记录，内容包含失败的完整命令字符串及返回码，随后继续下一轮循环。

---

## 6. 日志心跳保活

### 6.1 含义
心跳保活在此处的语义是自证存活：守护进程本身不对外提供网络接口或看门狗信号，而是通过每隔固定时间向自身日志文件写入一条信息级记录的方式，表明进程仍在正常运行。运维人员可通过查看日志文件中是否持续出现该记录来判断守护进程是否存活或已僵死/崩溃。

### 6.2 实现方式
- **周期**：固定 60 秒，与轮转周期完全解耦。
- **日志级别**：信息级。
- **内容**：包含醒目的前后缀标记，明确标识这是心跳日志。
- **机制**：采用毫秒级时间戳做减法判断，不使用计数器，因此即使系统时间发生跳变（如 NTP 同步），逻辑依然正确。

---

## 7. 进程退出条件

该守护进程没有设计任何正常的内部退出路径：
- 主循环条件为永真，除非收到外部信号或发生不可恢复的致命错误，否则将无限运行下去。
- logrotate 执行失败仅记录警告，不触发退出。
- 配置文件缺失或解析异常仅回退到默认值，不触发退出。
- 主函数末尾的返回语句在逻辑上不可达。

因此，该进程的生命周期完全由外部系统环境控制（如 systemd、init 脚本或手动 kill）。

---

## 8. 关键依赖与边界行为

| 场景 | 行为 |
|------|------|
| 配置环境变量未设置 | 使用默认路径作为配置目录 |
| 守护进程自身配置文件不存在 | 全部使用默认值，输出错误提示 |
| 配置文件某个键缺失/解析失败 | 该键使用默认值，输出错误提示 |
| 轮转周期配置为 0 或负数 | 强制修正为 30 分钟 |
| 日志级别越界 | 强制修正为 Info 级 |
| logrotate 配置文件不存在 | 由外部 logrotate 命令自行报错，守护进程记录警告后继续运行 |
| logrotate 命令执行失败 | 记录警告日志，继续循环，不退出 |
