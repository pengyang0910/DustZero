# localization 具体代码逻辑

> 本文档描述 localization（AMCL 定位进程）的内部代码逻辑，包括启动流程、双线程模型、定位主循环、粒子滤波器、传感器模型、初始位姿、重定位与位姿发布，不含源码。

---

## 1. 启动流程

### 1.1 信号注册
进程启动时注册中断信号处理函数，用于优雅退出。

### 1.2 实例化与配置加载
创建 AMCL 主类对象，在构造函数中完成以下初始化：
- 从配置文件加载全部参数（粒子数、噪声模型参数、激光模型类型、里程计模型类型、阈值等）。
- 初始化日志系统、互斥锁、各类状态标志。
- 设置里程计时间滤波器的缓存长度。
- 初始化各种内部数据结构（地图指针、粒子滤波器指针、传感器对象指针等置空）。

### 1.3 Start 调用
主线程调用启动方法，触发：
- 里程计和位姿的初始值清零。
- 在接收 LCM 实例上订阅各类消息通道，包括激光数据、地图数据、初始位姿命令、机器人状态、调试命令、RPC 命令等，所有订阅队列容量设为 1。
- 启动定位主循环线程和位姿发布循环线程。

### 1.4 主线程休眠循环
主线程进入 200ms 周期的休眠循环，仅监听退出信号。

### 1.5 Stop 调用
收到退出信号后，依次取消所有 LCM 订阅，等待两个工作线程结束，释放地图和粒子滤波器相关内存。

## 2. 双线程模型

进程采用严格的双线程架构，职责分离如下：

### 2.1 mThread（定位主循环线程）
- **职责**：处理所有 LCM 消息回调、执行粒子滤波更新、地图请求、激光与里程计数据融合。
- **运行节奏**：每周期先处理接收实例上所有待处理消息（通过非阻塞轮询直到队列空），然后休眠补足至约 20ms 周期。
- **绑核**：线程启动后绑定到指定的 CPU 核心。

### 2.2 pThread（位姿发布循环线程）
- **职责**：在发布 LCM 实例上订阅里程计消息（用于自身接收原始里程计），并周期性执行位姿发布函数。
- **运行节奏**：每 20ms 执行一次，先处理发布实例上的待处理消息，再发布位姿。
- **绑核**：同样绑定到指定 CPU 核心。

### 2.3 数据交互方式
- **激光数据**：通过接收实例的回调直接写入激光成员变量，受互斥锁保护。
- **里程计数据**：通过发布实例的回调写入时间滤波器和里程计数据，受独立互斥锁保护。
- **AMCL 位姿结果**：定位线程在粒子滤波更新后计算出的位姿受独立互斥锁保护，发布线程读取并发布。
- **时间对齐**：定位线程在处理激光消息时，通过时间滤波器查询与激光时间戳对齐的里程计位姿。

## 3. 定位主循环逻辑

### 3.1 LaserScan 数据处理流程
激光消息到达时触发回调，处理流程如下：

#### 数据拷贝与有效性检查
将激光数据拷贝到内部缓冲区，检查地图是否已加载、俯仰角是否有效。

#### 激光校正
如果配置启用激光校正，对右侧远距离激光点进行对数衰减校正。

#### 激光器注册
若该激光器首次出现，基于激光安装外参在粒子滤波器中注册一个新的激光传感器实例，并记录帧名到索引的映射。

#### 时间对齐与里程计查询
- 以激光消息的时间戳为基准，调用时间滤波器查询对应时刻的里程计位姿。
- 若时间误差超过 100ms，则丢弃该帧激光数据。

#### 运动阈值判断
- 计算当前里程计位姿与上一次滤波更新位姿的差值。
- 若位移超过阈值或角度超过阈值，或强制更新标志置位，则标记该激光器需要更新；否则跳过。
- 若里程计跳变过大，则打印告警但继续处理（用于异常检测）。

#### 粒子滤波预测（动作更新）
- 若粒子滤波器未初始化，则使用初始位姿初始化，并强制标记所有激光器需要更新。
- 若已初始化且需要更新，则构造里程计动作数据，调用里程计模型对所有粒子执行位姿预测。

#### 观测更新（传感器更新）
- 构造激光数据，将激光距离值按最小和最大范围截断，小于最小阈值的设为最大范围值。
- 调用对应激光器实例的传感器更新函数，计算所有粒子的权重，并归一化。
- 记录总得分和有效激光点数。

#### 重采样
每次激光更新后递增重采样计数器，当达到重采样间隔的整数倍且满足条件时，触发重采样。

#### 粒子云发布
若非强制更新，则构造粒子云消息，包含当前样本集中所有粒子的位姿，通过 LCM 发布到粒子云通道。同时计算粒子云的均值、方差，用于收敛性监控。

### 3.2 Odom 数据处理流程
里程计消息通过发布实例的回调接收，处理如下：
- **坐标变换**：根据地图旋转角对原始里程计进行二维旋转，转换到 AMCL 地图坐标系。
- **中值滤波**：维护一个长度为 3 的里程计环形缓冲区，取角度的中值作为输出，用于抑制野点。
- **时间滤波器入队**：将中值滤波后的里程计数据按时间戳推入时间滤波器。
- **状态更新**：设置里程计更新标志；若粒子滤波器尚未初始化，则设置已收到初始位姿标志。

### 3.3 TimeFilter 时间对齐机制
时间滤波器是一个基于时间戳的插值/查找缓冲区：
- 里程计数据以时间戳为键入队，缓存长度为 10。
- 当处理激光消息时，以激光时间戳查询时间滤波器，返回最接近该时刻的里程计位姿，并给出时间误差。
- 若查找误差超过 100ms，判定为时间对齐失败，丢弃当前激光帧。

### 3.4 地图消息处理与坐标转换
地图通过地图更新回调接收：
- **来源过滤**：仅处理 caller 为 AMCL 的地图消息，避免处理其他模块发布的地图。
- **时间戳校验**：检查地图消息时间戳延迟，若超过 100ms 发出警告。
- **地图转换**：将栅格地图消息转换为内部地图格式，解析地图宽高和分辨率，计算地图原点，将数据字节解析为占据状态。
- **首次地图处理**：清空旧的激光器对象和地图内存；预计算自由空间索引列表，用于后续均匀采样初始化；根据当前位姿和初始协方差初始化粒子滤波器；创建里程计模型对象和激光传感器对象，根据配置设置对应模型参数。
- **地图更新监控**：主循环中持续比较 SLAM 地图更新时间戳和当前地图时间戳，若 SLAM 地图更新时间比当前地图新超过 1 秒，则自动发送地图请求。

## 4. 粒子滤波器内部逻辑

### 4.1 双样本集切换机制
粒子滤波器维护两个样本集，通过当前索引标识当前激活集：
- 所有操作（动作更新、传感器更新、重采样）均在当前激活集上进行。
- 重采样时，从当前集采样生成新集，然后切换当前索引指向新集。
- 初始化时，两个样本集均预分配最大粒子数大小的样本数组和 K-D 树空间。

### 4.2 K-D 树聚类与收敛性检测
每个样本集内置一个三维 K-D 树，空间分辨率为：
- X/Y 方向：0.05 米
- 角度方向：10 度

**聚类流程**：
- 每次重采样或初始化后，调用 K-D 树聚类函数。
- 将所有叶子节点加入队列。
- 采用连通区域标记法（26 邻域搜索，包含角度维的环绕处理），为每个叶子节点分配聚类标签。
- 角度维做特殊环绕处理。

**收敛性检测**：
- 计算当前样本集所有粒子在 X/Y 方向的均值。
- 若所有粒子与均值的 X/Y 偏差均不超过距离阈值，则标记该样本集及整个粒子滤波器为已收敛；否则标记为未收敛。

### 4.3 选择性重采样的触发条件
重采样函数的逻辑如下：
- **选择性重采样开关**：若开启选择性重采样，且当前集的有效粒子数大于样本数的一半，则跳过重采样，直接拷贝当前集到另一集并切换。
- **自适应 KLD 采样**：若需要重采样，则持续采样直到样本数达到计算出的下限。该下限基于 K-D 树的叶子数，使用 KLD 采样公式计算，结果限制在最小和最大粒子数之间。
- **随机注入（Augmented MCL）**：计算权重差，若小于 0 则置 0。每次采样时以该概率从全局均匀分布中生成新粒子，以互补概率按权重从旧集采样。若发生随机注入，重置慢速和快速权重平均值为 0。

## 5. 激光传感器模型

激光模型支持三种可选模型，通过配置文件切换：

### 5.1 BEAM 模型（射线模型）
- 对每个粒子，将激光坐标转换到世界坐标。
- 按最大光束数均匀下采样激光点。
- 对每个采样光束，从地图中射线追踪获取期望距离。
- 观测概率为四种成分的混合：
  - **命中项**：高斯分布，描述正确检测到障碍物的噪声。
  - **短距项**：指数分布，描述意外障碍物导致的短读数。
  - **最大距项**：处理未检测到障碍物的情况。
  - **随机项**：均匀分布的随机噪声。
- 所有光束的概率通过立方和方式组合，最终更新粒子权重。

### 5.2 LIKELIHOOD_FIELD（似然场模型）
- 预处理：计算地图中每个单元格到最近障碍物的距离（即似然场）。
- 对每个粒子，将激光端点转换到地图栅格坐标。
- 忽略最大距离读数和无效读数。
- 距离取自似然场，高斯方差根据观测距离自适应调整（远距离激光点容忍更大误差）。
- 概率为命中高斯项与随机项的混合，同样通过立方和组合。

### 5.3 LIKELIHOOD_FIELD_PROB（概率似然场 + Beam Skipping）
- 基于似然场模型，增加光束跳过机制。
- 仅在粒子滤波器已收敛时启用 beam skipping。
- 第一轮遍历所有粒子和光束，计算每束光的概率，同时统计有多少粒子在该光束上的障碍物距离小于阈值。
- 若某光束上超过阈值比例的粒子与地图一致，则标记该光束为有效；否则跳过。
- 若被跳过的光束比例超过错误阈值，说明滤波器可能收敛到错误位姿，此时回退为使用全部光束。
- 使用对数概率求和再取指数的方式组合，数值稳定性更好。

## 6. 里程计模型

里程计模型支持四种模型，通过配置文件切换：

### 6.1 DIFF（差分驱动模型）
将里程计增量分解为三个阶段：
- 从旧位姿到位移方向的初始旋转。
- 直线平移距离。
- 平移后的最终旋转。

对每个粒子，为三个量分别采样高斯噪声：
- 初始旋转的噪声方差与旋转平方加平移平方成正比。
- 平移的噪声方差与平移平方加旋转平方和成正比。
- 最终旋转的噪声方差与旋转平方加平移平方成正比。

将采样后的增量按旋转-平移-旋转顺序应用到粒子位姿上。对初始和最终旋转的噪声做对称化处理，使前进和后退运动噪声模型对称。

### 6.2 OMNI（全向驱动模型）
将增量分解为平移距离、旋转和侧移。对每个粒子，基于当前粒子位姿和增量方向计算运动方向角。分别采样平移、旋转和侧移的高斯噪声，应用到粒子位姿。

### 6.3 DIFF_CORRECTED / OMNI_CORRECTED
与 DIFF 和 OMNI 逻辑相同，但噪声标准差计算时取平方根（即标准差而非方差），使噪声模型更符合统计学定义。

## 7. 初始位姿设置

初始位姿命令通过专用回调处理：

### 7.1 数据保存
将接收到的位姿和协方差保存到临时消息。

### 7.2 地图检查
若尚未收到首张地图，则根据命令中的分布类型记录，并主动请求地图。

### 7.3 高斯分布初始化
- 释放旧粒子滤波器，重新分配新的粒子滤波器。
- 将命令中的位姿和协方差转换为假设结构，保存到初始位姿假设。
- 调用应用初始位姿函数，使用高斯分布重新初始化当前样本集的所有粒子。

### 7.4 均匀分布初始化（重定位模式）
- 释放旧粒子滤波器，重新分配最大粒子数为远大于正常值的重定位粒子数。
- 调用均匀分布位姿设置函数，将每个粒子在地图的自由空间内均匀随机采样位姿，权重均分。
- 重置 K-D 树、聚类统计，并将收敛标记清零。

### 7.5 里程计模型重置
无论哪种分布，都会重新创建里程计模型对象，并将模型参数临时设为较大值，以允许重定位期间更大的运动不确定性。

## 8. 重定位逻辑

重定位在系统中通过均匀分布初始化模式触发，具体行为如下：
- **粒子发散**：所有粒子在地图自由空间内均匀随机分布，覆盖全图可能位姿。
- **大粒子数**：使用远大于正常模式的重定位粒子数。
- **高里程计噪声**：临时增大里程计模型参数，允许粒子在预测阶段快速扩散。
- **收敛检测**：每次重采样后通过收敛检测函数检测粒子是否聚集在足够小的区域内。
- **重定位完成回调**：收到外部调试命令数据为特定值时，判定为重定位完成，设置标志，在下次里程计回调时恢复标准里程计模型参数。

## 9. 位姿发布逻辑

位姿发布在发布线程中执行，周期约 20ms：

### 9.1 未初始化阶段的发布
若粒子滤波尚未输出有效位姿，则直接发布当前最新原始里程计作为 AMCL 里程计，位姿值与原始里程计相同。

### 9.2 初始化后的位姿插值
若已有有效 AMCL 位姿，则进行里程计增量插值：
- 获取时间滤波器最新的原始里程计和上一次滤波时的里程计。
- 若时间差超过阈值，计算当前与上次里程计的位姿差：
  - 角度差直接累加到 AMCL 位姿。
  - 位移差计算后，根据 AMCL 当前朝向角度将其分解到 X/Y 方向累加。
  - 若位移的前进方向与当前朝向夹角大于 90 度，则位移取负（处理后退运动）。
- 更新上次里程计为当前里程计。

### 9.3 消息构造与发布
- **AmclPose**：包含位姿、协方差矩阵、权重、时间戳，通过发布实例发布。
- **AmclOdom**：将 AMCL 位姿再做一次固定角度的坐标变换后发布，同时计算得分（平均每个激光点的匹配得分）。
- **AmclPoseInfo**：包含命中得分、激光点数、得分比率、状态字符串。
- **AmclParticleCloud**：在定位线程中每次激光更新后发布当前所有粒子位姿。

## 10. HackCmd 和 RpcCmd 的处理

### 10.1 HackCmd（调试/遥操作命令）
通过调试命令回调接收，支持多种手动调试操作：
- **位姿微调**：对当前粒子云的所有粒子整体施加固定偏移（X/Y 平移、旋转），用于手动校正位姿漂移。
- **参数动态调整**：在线增减里程计模型的 alpha 参数，实时改变运动预测的不确定性。
- **功能开关**：启用/禁用参数自适应校正和粒子云前向校正。
- **异常命令**：接收来自 SLAM 模块的异常通道消息，用于通知 AMCL 重定位已完成，恢复标准里程计参数。

### 10.2 RpcCmd（远程过程调用命令）
通过 RPC 命令回调接收，支持系统级控制：
- **进入睡眠**：收到特定命令时直接结束进程。
- **清洁任务标记**：清洁开始时设置标志并在日志文件中记录时间戳；清洁状态会影响里程计模型的参数恢复逻辑。

### 10.3 机器人状态处理
通过机器人状态回调接收：
- **碰撞检测**：记录左右碰撞传感器触发时间戳。
- **俯仰角监控**：基于四元数计算俯仰角，前若干样本取均值作为初始基准。若后续俯仰角与基准差超过阈值且持续超过多个周期，则设置俯仰角无效标志，此后所有激光数据被丢弃，直到俯仰角恢复稳定。

### 10.4 粒子云前向校正
在激光更新后的定位线程中，若启用前向校正：
- 计算当前 AMCL 位姿的前向命中得分，通过在特定激光角度范围内进行射线追踪，统计激光端点与地图障碍物的匹配程度。
- 若前向偏差绝对值大于阈值，则计算一个与偏差平方成正比的高斯噪声，对所有粒子的 X/Y 位置沿朝向方向施加随机偏移，实现前向漂移的在线校正。旋转期间跳过该校正。

### 10.5 参数自适应校正
在激光更新后的定位线程中，若启用参数自适应校正：
- 计算激光命中得分，即激光端点通过射线追踪与地图障碍物的匹配百分比。
- 若命中得分低于阈值且里程计模型尚未切换，则临时将 alpha 参数设为较大值，增大预测不确定性以帮助粒子恢复。
- 若命中得分高于阈值且已切换，则恢复为标准配置参数。
